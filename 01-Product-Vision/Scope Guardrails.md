# Scope Guardrails

## Active

- Technical documentation cloning
- Source document intake
- Intent classification for technical documentation
- Template-based output generation
- AI rules, hooks, context, memory, and prompt organization

## Deferred

- Functional documentation
- Peer review workflow
- Sora deck creator
- Multi-template marketplace

## Rule

Do not build future skills inside the active technical documentation workflow. Keep extension points visible, but keep execution paths narrow until a skill is promoted from roadmap to active scope.
