# Technical Doc Quality Bar

Generated technical documentation should be:

- Source-grounded: every major claim comes from the uploaded source, durable context, or an explicit assumption.
- Structured: output follows the approved technical documentation template.
- Reviewable: open questions, risks, assumptions, and unresolved source gaps are visible.
- Implementable: engineers can identify systems, dependencies, interfaces, steps, and failure modes.
- Maintainable: decisions and context are separated from one-off draft text.

## Minimum Required Sections

- Summary
- Source Inputs
- Architecture or Technical Context
- Data and Integration Details
- Implementation Plan
- Operational Concerns
- Risks and Assumptions
- Open Questions
- Review Checklist
