## 1. Summary

Describe the technical purpose, target system, and intended outcome.

## 2. Source Inputs

List uploaded files, references, and extracted source facts used to create this document.

## 3. Technical Context

Capture the current system, business process dependency, technical constraint, and target change.

## 4. Architecture

Describe components, boundaries, interactions, and ownership.

## 5. Data and Integration Details

Document APIs, schemas, files, events, batch jobs, interfaces, transformations, and validation rules.

## 6. Implementation Plan

Break the work into concrete technical steps.

## 7. Operational Concerns

Cover deployment, monitoring, logging, alerting, rollback, support, and known failure modes.

## 8. Security and Compliance Notes

Capture access, secrets, sensitive data, retention, auditability, and control considerations.

## 9. Risks and Assumptions

Separate source-backed risks from AI-generated assumptions.

## 10. Open Questions

List unresolved questions that block implementation, review, or release.

## 11. Review Checklist

- [ ] Source document is linked.
- [ ] Output follows this template.
- [ ] Assumptions are labeled.
- [ ] Open questions are visible.
- [ ] Technical reviewer is assigned.
