Overview

Transform raw files into ready docs.

Control Library turns a dropped-in source file into a structured document by routing it through a chosen skill, matched reference guidance, and a gold-standard template — entirely on your own machine.

Core Principles

Skill-agnostic by design

The Control Library classifies intent, routes to the right registered skill, and returns structured output without locking the pipeline to one document type.

1

Route by intent

Developer requests are matched to the skill that fits the work.

2

Keep skills independent

Each skill owns its behavior without changing other skills.

3

Standardize output

References, templates, rules, and checks keep results repeatable.

4

Extend safely

New skills can be added without rebuilding the other layers.

Quick Start

Run a document through the library

Requires: Codex CLI, installed and authenticated locally — Run Skill.bat calls codex exec under the hood.

1

Drop in a source file

Place a supported text-readable file into 1 inbound/.

2

Run the harness

Double-click Run Skill.bat to start the local workflow.

3

Choose a skill

Select the Markdown skill that describes the transformation you want.

4

Review the output

Find generated Markdown in 6 output/; the source moves to 1 inbound/Done/.

Add Skill

How to add a new skill

Use the CreateSkill skill when the library needs a new reusable document behavior.

1

Run the harness

Double-click Run Skill.bat, then choose the CreateSkill skill.

2

Answer the prompts

Provide the skill name, goal, functionality, expected input, and expected output.

3

Review the file

The new Markdown skill is created in 3 skills/ using your chosen name.

4

Run it

Drop a source file into 1 inbound/, run the harness, and select the new skill.
